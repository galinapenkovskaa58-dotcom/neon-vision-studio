## Цель

Заменить текущую сетку карточек портфолио на более «живое» отображение: каждое портфолио — это **цветной кругляшок**, к которому ведёт **извилистая стрелка**. При наведении сверху веером раскрываются превью фото, кругляшок подсвечивается неоновым свечением (как бейдж «НЕЙРОФОТОСЕССИИ» в шапке). Клик — открывает текущий лайтбокс с фотографиями.

## Что меняется / что НЕ меняется

- Меняется ТОЛЬКО публичный компонент `src/components/landing/Portfolio.tsx` (визуальное отображение списка).
- НЕ меняется: админка, БД, лайтбокс (`PortfolioLightbox`), focal points, мозаика для админ-превью (`PortfolioCard` остаётся — используется в админ-предпросмотре).

## Дизайн

### Кругляшок (node)
- Круг ~110–130px, неоновая обводка + мягкое свечение цветом из палитры: `neon-cyan`, `neon-purple`, `neon-pink`, `neon-blue` — циклически по индексу.
- Внутри — миниатюра первой фотографии портфолио (object-cover с сохранёнными `image_positions[0]`), затемнение + название категории/тайтла мелким текстом снизу или по центру при hover.
- При hover: усиленный glow (`box-shadow` с двумя слоями того же цвета — аналогично бейджу `cyan: shadow-[0_0_20px_hsl(var(--neon-cyan)/0.25)]`, но сильнее: 0_0_30px / 0_0_60px), лёгкое `scale(1.05)`.

### Раскладка и стрелки
- Кругляшки расположены в «змейке» по 3–4 в ряд, со смещениями по вертикали (каждый второй ниже/выше) — создаёт ощущение пути.
- Между соседними кругляшками рисуется **извилистая SVG-стрелка** (cubic Bezier `M x1 y1 C ...`), цвет — градиент из цвета текущего кругляшка в цвет следующего, с маленьким наконечником (`marker-end`).
- Один общий `<svg>` оверлей абсолютно позиционирован поверх контейнера; координаты якорей кругляшков считаются через `useRef` + `getBoundingClientRect` относительно контейнера + `ResizeObserver` для пересчёта при ресайзе.
- На мобильных (sm-) — стрелки прячутся, кругляшки выстраиваются в один столбец (стрелки между ними — простые вертикальные изгибы или скрыты).

### Веер превью при hover
- При наведении на кругляшок над ним появляется «веер» из первых 5 фотографий портфолио.
- Каждая мини-карточка (~80×100px) — `motion.div` с поворотом `-30°…+30°` равномерно, выезжает из центра кругляшка вверх с `scale 0.6 → 1`, `y 0 → -90px`, лёгкая задержка по индексу (`stagger 0.04s`).
- Веер pointer-events: none, чтобы клик/hover не «прыгали».
- Фон превью — стеклянная плашка (`glass`), тонкая обводка цветом кругляшка.

### Клик
- Открывает существующий `PortfolioLightbox` (логика `openItem` сохраняется как сейчас).

## Технические детали

- В `Portfolio.tsx`:
  - Убрать рендер `PortfolioCard` в гриде, оставить только заголовок секции и фильтр по категориям.
  - Добавить новый внутренний компонент `PortfolioNode` (или вынести в `src/components/portfolio/PortfolioNode.tsx`) — круг + hover-веер.
  - Добавить `PortfolioPath` — SVG-оверлей со стрелками, считающий координаты центров узлов.
- Цвета по индексу: `const palette = ['cyan','purple','pink','blue'] as const;` → классы glow и stroke выбираются мапой.
- Анимации: `framer-motion` (`motion.div`, `AnimatePresence` для веера на hover).
- Доступность: каждый кругляшок — `<button>` с `aria-label={item.title}`, фокус-кольцо неонового цвета.

## Структура раскладки (пример для desktop)

```text
 (1)─~~~↘
         (2)─~~~↘
                 (3)
 ↗~~~─(6)        │
        ↖~~~─(5)─~~~↙
                (4)
```

Координаты узлов задаются через CSS grid с принудительными `row-start/col-start` + небольшой `translateY` для волны; стрелки отрисовываются по фактическим центрам.
